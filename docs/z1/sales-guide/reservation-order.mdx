# 预订订单

## 业务场景

预订订单（预售）用于商品正式发售前的预售活动，会员支付订金锁定购买名额，正式发售后再支付尾款完成购买。适用于新品预售、限时预订等场景。

## 前置条件

- 已配置预售活动
- 预售商品已设置可预订数量（名额）
- 会员已完成注册并登录

## 名词解释

| 术语 | 说明 |
|------|------|
| 订金 | 预订时支付的小部分金额，用于锁定名额 |
| 尾款 | 发售后支付剩余款项 |
| 膨胀金 | 订金增值金额，可抵扣更多货款 |
| 预订商品 | 预售活动中的商品 |

## 预订规则

| 规则 | 说明 |
|------|------|
| 支付时限 | 下单后5分钟内完成支付，否则自动取消 |
| 限购数量 | 每笔订单只能购买1个预售商品 |
| 不锁货 | 预订订单不锁定库存 |
| 优惠限制 | 不支持优惠券、赠品、代金券、积分 |

## 订单状态

| 状态 | 说明 |
|------|------|
| 待支付 | 预订成功，等待支付订金 |
| 已支付 | 订金支付成功，等待尾款 |
| 已完成 | 尾款支付完成 |
| 已取消 | 超时未支付或主动取消 |
| 已退款 | 订金已退还 |

## 操作步骤

### 会员预订商品

1. 进入预售商品详情页
2. 点击"立即预订"
3. 确认订单信息
4. 完成订金支付（5分钟内）

### 支付尾款

1. 收到尾款支付通知
2. 进入预订订单页面
3. 点击"支付尾款"
4. 确认并完成支付

### 追加连带商品

职员可以在预订订单上追加其他商品：

1. 打开预订订单详情
2. 点击"追加商品"
3. 选择要添加的商品
4. 确认后生成销售订单

### 修改处理部门

1. 仅当前处理部门职员可修改
2. 订单状态需为"待处理"
3. 选择新的处理部门

## 金额计算

```
应付金额 = 实际价格 - 订金 - 膨胀金
```

| 说明 | 示例 |
|------|------|
| 订金 | 100元 |
| 膨胀金 | 500元 |
| 预订商品价格 | 500元 |
| 尾款 | 500 - 100 - 500 = -100 → 0元 |

## 常见问题

### 预订后可以取消吗？

可以。未支付尾款前可取消，订金将退还。

### 超时未支付怎么办？

下单后5分钟内未完成支付，系统自动取消订单。

### 预订商品可以更换吗？

可以更换为其他任意分类商品，但必须保留一个预订商品。
