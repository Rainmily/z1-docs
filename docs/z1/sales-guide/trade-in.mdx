# 以旧换新

## 业务场景

以旧换新是一种营销手段，顾客购买新机时同时回收旧手机，可享受换机补贴。通过将零售单与回收单关联，统计零售商品的回收转化率。

## 前置条件

- 已完成一笔回收单
- 回收单状态为"待发货"及之后
- 顾客手机号与回收单一致（系统设置）

## 基本规则

| 规则 | 说明 |
|------|------|
| 必须购买新机 | 以旧换新必须同时有新机零售单 |
| 必须回收旧机 | 必须有一笔回收单关联 |
| 换机补贴 | 使用换新补贴类型卡券减免新机款 |

## 关联限制

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 必须同一手机号 | 是 | 零售单与回收单顾客手机号一致 |
| 必须同一门店 | 是 | 零售单与回收单门店一致 |
| 最小间隔天数 | 1天 | 回收单付款与零售单时间间隔 |
| 允许取消关联 | 否 | 是否可手动取消关联 |

## 操作步骤

### 关联回收单

1. 创建/打开零售单
2. 点击"关联回收单"
3. 选择符合条件的回收单
4. 确认关联

### 使用换机补贴

1. 关联回收单后
2. 系统自动显示可用换新补贴
3. 选择使用换新补贴卡券
4. 系统自动计算减免金额

### 取消关联

1. 打开已关联的零售单
2. 点击"取消关联"
3. 确认取消（需系统设置允许）

### 退货后处理

零售单退货退款后，必须手动取消关联才能释放回收单。

## 回收跟机率

```
回收跟机率 = SPU零售单关联回收数量 ÷ (SPU零售单销售数量 - 退货数量)
```

## 常见问题

### 关联失败怎么办？

检查是否满足关联条件：
- 回收单是否已被其他订单关联
- 手机号/门店是否一致
- 时间间隔是否满足

### 换机补贴如何使用？

必须先关联回收单，才能使用换新补贴卡券。

### 一笔回收单能用几次？

只能被成功关联一次。
