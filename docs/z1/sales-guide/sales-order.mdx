# 销售订单

## 业务场景

销售订单功能用于管理零售销售业务，记录商品销售信息，支持多种支付方式和优惠应用。系统支持 PC 端和移动端下单。

## 前置条件

- 商品已有库存
- 会员信息已录入（如需关联会员）
- 收银员已登录系统

## 订单类型

| 类型 | 说明 |
|------|------|
| 零售单 | 门店日常零售业务 |
| 网销单 | 小程序商城订单 |
| 批发单 | 批发业务订单 |

## 名词解释

| 术语 | 说明 |
|------|------|
| 订单编号 | 每笔订单的唯一标识 |
| 实付金额 | 实际支付金额 |
| 优惠金额 | 优惠抵扣金额 |
| 订单状态 | 订单当前所处阶段 |

## 订单状态

| 状态 | 说明 |
|------|------|
| 待付款 | 订单创建等待支付 |
| 已付款 | 支付完成 |
| 已发货 | 商品已发出 |
| 已完成 | 交易完成 |
| 已取消 | 订单已取消 |
| 已退款 | 已进行退款处理 |

## 操作步骤

### 创建销售订单

1. 进入 **销售管理** → **销售订单**
2. 点击 **新建订单**
3. 选择订单类型

#### 选择商品

1. 扫描商品条码或搜索商品
2. 选择商品数量
3. 确认商品信息

#### 选择支付方式

| 支付方式 | 说明 |
|----------|------|
| 现金 | 现金支付 |
| 银行卡 | 银行卡支付 |
| 微信支付 | 微信扫码/条码支付 |
| 支付宝 | 支付宝扫码/条码支付 |
| 会员储值 | 使用会员储值金支付 |
| 混合支付 | 多种方式组合支付 |

#### 应用优惠

1. 系统自动显示可用优惠
2. 选择优惠方式
3. 确认优惠金额

#### 完成订单

1. 确认订单信息
2. 完成支付
3. 打印小票

### 订单管理

#### 查看订单

1. 输入订单编号或选择日期范围
2. 点击查询
3. 查看订单详情

#### 退款/退货

1. 找到需要退款的订单
2. 点击退款按钮
3. 选择退款商品和退款原因
4. 确认退款金额
5. 完成退款操作

### 订单统计

| 统计项 | 说明 |
|--------|------|
| 销售总额 | 当日/当月销售总额 |
| 订单数量 | 当日/当月订单数 |
| 客单价 | 平均每单金额 |
| 退货率 | 退货订单占比 |

## 常见问题

### 如何修改订单？

订单在未完成前可以修改，完成后如需修改需进行退款重开。

### 混合支付如何操作？

选择多种支付方式，系统会按选择的金额比例进行拆分支付。

### 退款多久到账？

根据支付方式不同，退款到账时间有所不同：现金当场退还，线上支付1-7个工作日到账。
