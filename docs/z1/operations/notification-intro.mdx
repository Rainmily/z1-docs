# 通知模块介绍

## 业务场景

通知模块是通知中心的后端支撑，提供通知发送、接收、阅读状态跟踪等基础能力。支持多种通知渠道，确保用户及时收到重要信息。

## 前置条件

- 消息通道已配置
- 通知模板已创建

## 通知类型

| 类型 | 说明 | 发送渠道 |
|------|------|----------|
| 系统通知 | 系统推送的重要消息 | 应用内/短信 |
| 业务通知 | 业务流程触发 | 应用内/企微 |
| 营销通知 | 营销活动推送 | 短信/企微 |

## 通知结构

| 字段 | 说明 |
|------|------|
| 通知ID | 通知唯一标识 |
| 通知标题 | 通知主题 |
| 通知内容 | 通知正文 |
| 发送时间 | 发送时间戳 |
| 接收人 | 通知目标用户 |
| 阅读状态 | 已读/未读 |

## 发送渠道

| 渠道 | 说明 |
|------|------|
| 应用内通知 | 系统内消息中心 |
| 短信 | 手机短信 |
| 企业微信 | 企微消息推送 |
| 邮件 | 邮箱通知 |

## 通知模板

系统支持预设通知模板，包含：
- 模板变量（可动态替换）
- 发送渠道配置
- 发送时间规则
