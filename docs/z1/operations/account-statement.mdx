# 往来对账单

## 业务场景

往来对账单面向企业内外部用户，提供实时、准确、可交互的往来交易核对工具，是连接业务系统与财务结算的关键枢纽。

## 前置条件

- 有往来单位（客户/供应商）数据
- 有交易记录

## 名词解释

| 术语 | 说明 |
|------|------|
| 往来单位 | 对账对象（客户=应收，供应商=应付） |
| 对账单 | 特定期间内的交易明细及汇总 |
| 会计时间 | 对账统计周期 |
| 期初 | 对账周期开始时间 |
| 期末 | 对账周期结束时间 |

## 计算规则

### 期初余额

```
应收款/应付款 = 期初余额
```

### 本期发生

```
应收增加 - 应收减少 = 本期应收
应付增加 - 应付减少 = 本期应付
```

### 期末余额

```
期末应收 = 期初应收 + 本期应收
期末应付 = 期初应付 + 本期应付
```

## 显示规则

| 情况 | 显示方式 |
|------|----------|
| 正数 | 应收款显示金额，应付款为空 |
| 负数 | 应付款显示金额，应收款为空 |
| 零 | 应收款显示0，应付款为空 |

## 操作步骤

### 查看对账单

1. 进入 **财务管理** → **往来对账单**
2. 选择往来单位
3. 选择对账期间
4. 查看对账单明细

### 核对交易

1. 查看交易明细列表
2. 核对每笔交易金额
3. 如有差异，记录调整

## 常见问题

### 为什么显示负数？

负数表示应付款大于应收款，系统只显示正数方。

### 没有发生的往来如何显示？

应收款、应收增加、应收减少、应付增加、应付减少均记0.00。
