# 通知中心

## 业务场景

通知中心统一管理系统内所有需要用户处理的事项，包括待办事项、抄送知会等，方便用户快速查看和处理各类业务通知。

## 前置条件

无

## 名词解释

| 术语 | 说明 |
|------|------|
| 通知 | 系统推送的消息提醒 |
| 收件人 | 通知的接收对象 |
| 抄送人 | 被知会的对象 |
| 业务类型 | 通知的分类 |

## 通知结构

| 字段 | 说明 |
|------|------|
| 通知时间 | 通知创建时间 |
| 收件人 | 收到通知的人（可多选） |
| 抄送人 | 被抄送的人 |
| 通知标题 | 50字以内 |
| 通知内容 | 2000字以内，支持Markdown |
| 操作链接 | 可配置跳转链接 |

## 通知状态

| 状态 | 说明 |
|------|------|
| 未读 | 用户未查看，有醒目提示 |
| 已读 | 用户已点击查看 |

## 通知列表

| 类型 | 说明 |
|------|------|
| 我收到的 | 当前用户为收件人 |
| 抄送我的 | 当前用户为被抄送人 |

## 操作步骤

### 查看通知

1. 进入 **通知中心**
2. 选择通知类型（收到/抄送）
3. 查看通知列表

### 处理通知

1. 点击通知查看详情
2. 根据操作链接跳转到对应业务
3. 完成后续操作

### 标记已读

1. 点击"标为已读"
2. 或全部标为已读

## 常见问题

### 通知可以转发吗？

不可以，通知只发送给指定收件人和抄送人。

### 移动端支持操作吗？

移动端暂不支持点击通知操作跳转，需到PC端处理。
