# 部门模块

## 业务场景

部门模块用于管理企业组织架构，方便对职员进行分组管理。每个门店对应一个部门，但部门不一定是门店。

## 前置条件

无

## 名词解释

| 术语 | 说明 |
|------|------|
| 部门 | 企业组织架构的基本单位 |
| 上级部门 | 部门的层级归属 |
| 部门类型 | 部门的分类（如门店、职能等） |
| 价格类型 | 部门的定价策略 |

## 基本规则

- 部门最大数量：1000个
- 每个门店必须是一个部门
- 上级部门如有过会计分录则不能修改

## 操作步骤

### 新增部门

1. 进入 **系统设置** → **部门管理**
2. 点击 **新增部门**
3. 填写部门名称
4. 选择上级部门
5. 设置部门类型
6. 保存

### 修改部门

1. 找到要修改的部门
2. 点击编辑
3. 修改部门信息
4. 保存

### 停用部门

1. 找到要停用的部门
2. 点击停用
3. 确认停用

### 部门类型管理

| 功能 | 说明 |
|------|------|
| 新增部门类型 | 创建新的部门分类 |
| 修改部门类型 | 编辑类型信息 |
| 设置价格类型 | 配置部门的定价策略 |

## 常见问题

### 为什么限制1000个部门？

为了优化选择部门功能的体验，使用一次获取数据的方式。

### 门店必须是部门吗？

是的，每个门店必须是对应一个部门。

### 上级部门可以修改吗？

如果上级部门有过会计分录，则不能修改。
