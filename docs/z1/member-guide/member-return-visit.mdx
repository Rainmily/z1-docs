# 会员回访

## 业务场景

会员回访功能用于提醒专属导购对会员进行定期回访沟通，主要场景包括：

- **订单回访**：订单销售后一段时间进行回访
- **换机回访**：基于购机时间提醒会员换机

回访功能帮助门店提升客户服务质量，促进二次销售。

## 前置条件

- 会员有有效的销售订单
- 会员已绑定专属导购

## 名词解释

| 术语 | 说明 |
|------|------|
| 会员回访 | 程序进行订单记忆，到期自动创建回访 |
| 订单回访 | 针对订单内容进行的回访提醒 |
| 换机回访 | 针对订单购机时间进行的换机回访 |
| 回访人 | 客户的专属导购（无专属导购时由系统客服处理） |

## 回访限制

- 一个"会员回访"有且只有一个订单回访类型的会员回访记录
- 一个"会员回访"有且只有一个换机回访类型的会员回访记录

## 回访类型

| 类型 | 触发条件 |
|------|----------|
| 订单回访 | 零售单/商城订单中至少包含一台主营商品，且单次消费1000元以上，7天后提醒 |
| 换机回访 | 零售单/商城订单中至少包含一台主营商品，365天后提醒 |

### 回访触发说明

- 订单回访提醒时间：7天（每天读取历史数据，应该创建却没有被创建回访的订单，创建会员回访）
- 换机回访提醒时间：365天（每天读取历史数据，创建会员回访）
- 针对每笔订单均进行换机回访操作，避免用户流失

## 回访记录

每次与顾客沟通时，都需要添加回访记录：

| 字段 | 说明 |
|------|------|
| 记录文字 | 回访沟通内容 |
| 记录人 | 回访操作人员 |
| 记录时间 | 回访时间 |
| 回访方式 | 电话/短信/微信/企微/QQ/其他 |

## 回访状态

| 状态 | 说明 |
|------|------|
| 待回访 | 应当回访但无任何回访记录 |
| 进行中 | 开始回访操作后 |
| 回访失败 | 回访未能成功，需等待后重试 |
| 回访中止 | 回访失败超过限制次数后中止 |
| 回访成功 | 完成回访并确认 |

## 状态机

```
待回访 → 进行中 → 回访成功
              ↓
         回访失败 → 进行中（24小时后）
              ↓
         回访中止
```

## 回访规则

| 规则 | 说明 |
|------|------|
| 操作限制 | 回访可操作次数有限制（系统设置） |
| 时间限制 | 失败后24小时后方可重试 |
| 人员限制 | 仅专属导购可进行回访 |
| 失败次数 | 每个订单24小时限制情况仅会出现2次 |

### 回访失败处理

- 回访失败后需等待24小时才能再次回访
- 每个订单最多失败2次
- 超过限制后状态变为"回访中止"

### 回访成功

填写回访备注并手动确认后，状态变为"回访成功"。

## 回访单号

生成规则：`日期（年月日）+ 序号（五位00001）+ 回访单识别号`

| 识别号 | 类型 |
|--------|------|
| 01 | 订单回访 |
| 02 | 换机回访 |

## 常见问题

### 没有专属导购的会员由谁回访？

由系统设置中的客服人员组任意一人处理。

### 回访可以操作几次？

受系统设置的"回访可操作次数限制"控制。
