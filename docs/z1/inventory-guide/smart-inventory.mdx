# 智能库存

## 业务场景

智能库存通过数据分析自动建议补货和调拨，优化库存水平。系统根据仓库库存、销售数据自动计算最优配货方案，提高门店货物调配效率。

## 前置条件

- 历史销售数据充足
- 库存预警规则已配置
- 仓库、仓库组、总仓、中心仓关系已配置

## 名词解释

### 仓库体系

| 术语 | 说明 |
|------|------|
| 仓库 | 仓库的最小单位，通常是一个门店仓库 |
| 仓库组（蜂窝） | 由若干仓库组成，归属于同一级下的子仓库 |
| 中心仓 | 仓库组内可指定一个仓库作为中心仓，每个仓库组最多一个 |
| 总仓 | 负责给当前分类及子分类的中心仓分货、调拨，可存在多个 |
| 仓库大区 | 一个总仓及其下面的所有仓库组被称为仓库大区 |

### 库存状态

| 术语 | 说明 |
|------|------|
| 出库中库存 | 未完成的调拨出库单、已锁货未完成的销售订单，不参与智能配货 |
| 入库中库存 | 未完成的调拨入库单，不参与智能配货（不包含销售退） |
| 待售天数 | 库存数量和平均销售数量的比值，指剩余库存的可售天数 |

### 仓库层级结构

```
大区
├── 仓库1（总仓） // 一个大区内只允许存在1个总仓
├── 仓库组1
│   └── 仓库2（中心仓） // 一个仓库组内只允许存在1个中心仓
├── 仓库组2
│   ├── 仓库3（中心仓）
│   └── 仓库4
└── 仓库组3
    └── 仓库9
```

## 功能说明

### 库存预警

| 功能 | 说明 |
|------|------|
| 安全库存提醒 | 低于安全库存自动提醒 |
| 补货水位 | 根据待售天数设置补货警戒线 |

### 智能配货

根据仓库组内SKU待售天数计算配货数量：

- **仓库组SKU待售天数** = 仓库组内所有仓库某SKU库存数量 / 仓库组内所有仓库7天内平均销量
- 智能配货时，总仓根据各仓库组的待售天数分配货品

### 智能调拨

| 条件 | 说明 |
|------|------|
| 过滤条件 | 禁用的仓库不可作为总仓或中心仓 |
| 分货条件 | 根据待售天数计算各仓库应补货量 |

### 智能采货

根据采销设置和预定计划自动生成分货清单：

| 场景 | 说明 |
|------|------|
| 预定数量 ≤ 计划分货量 | 按预定比例分货，剩余按贡献率分货 |
| 预定数量 > 计划分货量 | 按预定先后顺序取货，剩余按贡献率分货 |

## 操作步骤

### 进入智能库存

1. 进入 **库存管理** → **智能库存**
2. 查看仓库体系概览

### 查看待售天数

1. 选择仓库组
2. 查看各SKU待售天数
3. 根据待售天数判断是否需要补货

### 智能配货操作

1. 总仓进入智能配货页面
2. 系统自动计算各仓库组分货方案
3. 确认分货数量
4. 生成调拨单

### 智能调拨操作

1. 进入智能调拨页面
2. 设置调拨条件（仓库组、SKU等）
3. 系统生成调拨建议
4. 确认后执行调拨

## 仓库禁用规则

| 仓库类型 | 禁用后表现 |
|----------|------------|
| 总仓禁用 | 分货时报错 |
| 中心仓禁用 | 分货时自动过滤 |

## 常见问题

### 一个仓库组可以有几个中心仓？

每个仓库组最多只能有1个中心仓。

### 总仓和中心仓可以是同一个仓库吗？

不可以，总仓和中心仓不允许是同一个仓库。

### 禁用的仓库可以参与配货吗？

禁用的仓库不可以作为总仓或中心仓，但可以被分配到仓库组（非中心仓）。
