# 采购功能

## 业务场景

采购功能用于管理门店商品采购业务，对接会计模块实现业务与财务数据打通，减少会计工作量，增加数据实时性。

## 前置条件

- 供应商信息已录入
- 仓库信息已配置

## 名词解释

| 术语 | 说明 |
|------|------|
| 采购订单 | 商品采购申请单据 |
| 往来单位 | 供应商信息 |
| 入库仓库 | 采购商品入库的仓库 |
| 采购单价 | 与供应商约定的采购价格 |

## 采购流程

1. 创建采购订单
2. 审批流程（可选）
3. 供应商发货
4. 商品入库
5. 财务结算

## 操作步骤

### 创建采购订单

1. 进入 **采购管理** → **采购订单**
2. 点击 **新建采购订单**
3. 填写订单信息
4. 添加采购商品
5. 确认采购金额
6. 提交审批

### 采购商品明细

| 字段 | 说明 |
|------|------|
| 商品名称 | 采购的商品 |
| 采购单价 | 供应商报价 |
| 库存单价 | 系统库存价格 |
| 采购数量 | 采购数量 |
| 合计金额 | 单价 × 数量 |
| 单价差额 | 采购单价 - 库存单价 |

### 审批模板

| 字段 | 说明 |
|------|------|
| 采购订单号 | 订单唯一标识 |
| 入库仓库 | 仓库编码+名称 |
| 往来单位 | 供应商信息 |
| 创建人 | 职员信息 |
| 采购总数量 | 所有商品数量合计 |
| 采购总金额 | 所有商品金额合计 |

## 常见问题

### 采购订单可以部分支付吗？

不可以，必须全额支付。

### 采购价格与库存价格有什么关系？

系统会显示单价差额，便于财务审核。
